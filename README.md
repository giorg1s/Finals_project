# WordPress CLI Manager Tool

## 1. Εισαγωγή & Σενάριο
[cite_start]Το παρόν εργαλείο αναπτύχθηκε στο πλαίσιο της εργασίας για τον ρόλο του **Τεχνικού Ανοιχτού Λογισμικού**[cite: 4]. [cite_start]Πρόκειται για ένα CLI (Command Line Interface) εργαλείο σε Python που αυτοματοποιεί την επικοινωνία με ένα τοπικό WordPress site (μέσω XAMPP) χρησιμοποιώντας το REST API[cite: 4, 14].

## 2. Θεωρητικό Υπόβαθρο & Ανοικτότητα (Μέρος Α)

### 2.1 Αδειοδότηση (Licensing)
[cite_start]Επιλέγω την άδεια **GPLv3**[cite: 10]. [cite_start]Η άδεια αυτή είναι απαραίτητη για τη διασφάλιση της ελευθερίας του κώδικα, καθώς επιβάλλει σε οποιονδήποτε τροποποιήσει ή διανείμει το λογισμικό να διαθέσει τον κώδικά του υπό την ίδια άδεια (copyleft), προστατεύοντας έτσι την ανοικτότητα της λύσης[cite: 10].

### 2.2 Επιχειρηματικό Μοντέλο (Managed Support)
[cite_start]Με βάση αυτό το εργαλείο, προτείνεται η παροχή υπηρεσιών **Managed Support** σε τρίτους[cite: 11]. Η πρόταση περιλαμβάνει:
* [cite_start]**Αυτοματοποιημένο Monitoring:** Χρήση του script για τον έλεγχο της κατάστασης του WordPress site του πελάτη[cite: 11].
* [cite_start]**Προληπτική Συντήρηση:** Παρακολούθηση των τελευταίων αναρτήσεων και διασφάλιση της ορθής λειτουργίας του API[cite: 11, 14].
* [cite_start]**Υποστήριξη:** Παροχή εξειδικευμένων τεχνικών υπηρεσιών με μηνιαία συνδρομή, επιτρέποντας στον πελάτη να εστιάσει στο περιεχόμενό του ενώ εμείς διαχειριζόμαστε την τεχνική αρτιότητα της πλατφόρμας[cite: 11].

### 2.3 Διαφάνεια
[cite_start]Η δημοσιοποίηση του κώδικα στο GitHub ενισχύει την αξιοπιστία του εργαλείου διότι επιτρέπει τον δημόσιο έλεγχο, τη συνεργατική ανάπτυξη και τη διασφάλιση ότι η λύση είναι ηθικά σωστή και τεχνικά άρτια[cite: 6, 12].

## 3. Διαχείριση Περιβάλλοντος (Μέρος Γ)

[cite_start]Για τη σωστή λειτουργία του εργαλείου, ακολουθήστε τα παρακάτω βήματα στο CMD[cite: 65, 66]:

1. **Δημιουργία εικονικού περιβάλλοντος:**
   ```cmd
   python -m venv venv
   [cite_start]``` [cite: 67]
2. **Ενεργοποίηση περιβάλλοντος:**
   ```cmd
   venv\Scripts\activate
   [cite_start]``` [cite: 68]
3. **Εγκατάσταση βιβλιοθήκης:**
   ```cmd
   pip install requests
   [cite_start]``` [cite: 69]
4. **Δημιουργία αρχείου εξαρτήσεων:**
   ```cmd
   pip freeze > requirements.txt
   [cite_start]``` [cite: 70]
